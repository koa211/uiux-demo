What is QA?

QA is a systematic process for preventing defects and verifying that software meets user needs. It blends prevention (process, standards) and detection (testing, reviews).

  • Functional
  • UI/UX
  • Accessibility
  • Performance
  • Security (basic)

The Test Pyramid

Balance speed and confidence by emphasizing fast, focused tests at the base.

  • Unit: Fast, isolated, verify functions/components.
  • Integration: Interactions between modules/services.
  • E2E: User-level flows across the system.

Bug Lifecycle

  1. New
  2. Triaged
  3. In-Progress
  4. Fixed
  5. Verified
  6. Closed

Tip: Separate severity (impact) from priority (when to fix).
